One month ago I proposed a nutty little system for keeping track of tasks: in a file called todo.txt. Later I published a set of scripts to manage todo.txt and asked for feedback.

Much to my delight, an army of plain text-lovin' lifehackers out there use todo.txt to keep themselves organized, and many of you sent in suggestions and improvements to my scripts in the comments and via email. I admitted from the get-go that my first attempt was half-baked and had more holes than Swiss cheese. Well, with all your generous help, dear readers, the Lifehacker todo manager has grown up to be super-strength and featureful.

I've compiled all the best reader-contributed updates and additions into one master todo script that adds, removes, edits, prioritizes, reprioritizes, lists and completes items in todo.txt. Let's take a look.

First, as per the advice of a wise and experienced bash scripter, instead of using different scripts to perform different actions on todo.txt, all the actions now get passed as a parameter to one master script, called todo. That means all your todo.txt work commands start with the word todo.

Let's walk through using the new system.

Example Usage

Once you've downloaded and enabled the todo script, to add an item to todo.txt, type the following on the command line (the dollar sign is the prompt; no need to type it):

$ todo add "notice the daisies"

To view your todo list, type:

$ todo list

1: notice the daisies

Edit this task like so:

$ todo replace 1 "smell the roses"

Now your list looks like:

$ todo list

1: smell the roses

To put this task in a project called "flowers" and add a context "outside," type:

$ todo append 1 "p:flowers @outside"

Here the "p:" indicates a project, and the "@" indicates the context. (Read more on the three project/context/priority axes of a sliceable, diceable task list.)

Advertisement

To view your todo list, type:

$ todo list

1: smell the roses p:flowers @outside

To add another task:

$ todo add "stop"

Now your todo list looks like:

$ todo list

1: smell the roses p:flowers @outside

2: stop

Then, to prioritize the roses as an A priority task:

$ todo pri 1 A

You'd get:

$ todo list

1: (A) smell the roses p:flowers @outside

2: stop

To prioritize stop as A, then the roses as B, you'd say:

$ todo pri 1 B

$ todo pri 2 A

And get:

$ todo list

2: (A) stop

1: (B) smell the roses p:flowers @outside

To add the stop task to the flowers project:

$ todo append 2 "p:flowers"

And todo.txt would look like:

$ todo list

2: (A) stop p:flowers

1: (B) smell the roses p:flowers @outside

Let's add a task for another project in todo.txt.

$ todo add "smell the coffee p:wakeup"

Our todo list now looks like:

$ todo list

2: (A) stop p:flowers

1: (B) smell the roses p:flowers @outside

3: smell the coffee p:wakeup

$ todo list p:flowers

2: (A) stop p:flowers @outside

1: (B) smell the roses p:flowers @outside

To mark the stop task as done:

$ todo do 2

And get:

$ todo list

1: (B) smell the roses p:flowers @outside

3: smell the coffee p:wakeup

2: x (A) stop p:flowers @outside

The x in front of the stop item means it's complete. To move out all your completed todo's to done.txt:

$ todo archive

Then your list looks like:

$ todo list

1: (B) smell the roses p:flowers @outside

2: smell the coffee p:wakeup

Getting Help

If your head hasn't spun off your shoulders yet, you see that there are tons of actions and options with the todo manager. When you're not sure what command you need to work with todo.txt in a certain way, simply type todo and help is on the way. It will look like this:

 $ todo USAGE: todo [ACTION] [OPTION]...  Actions:         add "THING I 
NEED TO DO p:project @context"           Adds TODO ITEM to your 
todo.txt.           Project and context notation optional.           
Quotes optional.          append NUMBER "TEXT TO APPEND"           Adds 
TEXT TO APPEND to the end of the todo on line NUMBER.           Quotes 
optional.          archive           Moves done items from todo.txt to 
done.txt.          del NUMBER           Deletes the item on line NUMBER 
in todo.txt.          do NUMBER           Marks item on line NUMBER as 
done in todo.txt.          list [TERM] [[TERM]...]           Displays 
all todo's that contain TERM(s) sorted by priority with line           
numbers.  If no TERM specified, lists entire todo.txt.          listpri 
[PRIORITY]           Displays all items prioritized PRIORITY.           
If no PRIORITY specified, lists all prioritized items.          pri 
NUMBER PRIORITY           Adds PRIORITY to todo on line NUMBER.  If the 
item is already            prioritized, replaces current priority with 
new PRIORITY.            PRIORITY must be an uppercase letter between A 
and Z.          replace NUMBER "UPDATED TODO"           Replaces todo on
 line NUMBER with UPDATED TODO.          report           Adds the 
number of open todo's and closed done's to report.txt.

Ready to give this baby a go? Here's how.

Installation


Note: Mac and Unix users, this will work in the Terminal for you. Windows peeps, you need to download and run the free Unix emulator, Cygwin, to use the todo script and a whole slew of other Unix utilities. Once Cygwin is installed, fire it up and do all the command line work at a Cygwin prompt.

Advertisement
  1. Right click and save the todo script to your computer. (Windows/Cygwin users, save it to your Cygwin directory.)
  2. Open todo in a text editor and change the lines with the .txt file paths in them to your file paths, ie, TODO_FILE="/Users/gina/Documents/todo/todo.txt", etc.
  3. Make the todo file executable like this:
  4. $ chmod 755 todo

Now you're ready to use it. First, back up your current todo.txt and done.txt. Then, try some of the examples above to get started. When you've got yourself a healthy todo.txt going on, don't forget that the Bird's eye report helps you do a regular progress review.
